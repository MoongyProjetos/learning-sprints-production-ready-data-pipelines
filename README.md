**Da experimentação à produção: pipelines de dados fiáveis, observáveis e auditáveis**

Este repositório acompanha a formação / tech talk **“Production-Ready Data Pipelines”**, parte do programa **Learning Sprints**, e tem como objetivo **demonstrar, de forma prática e orientada a arquitetura**, como evoluir pipelines de dados experimentais para soluções **robustas, auditáveis e preparadas para ambientes produtivos**, aplicando princípios de DataOps.

O foco está em **qualidade, automação, rastreabilidade e confiança nos dados**, através da adoção de práticas de **versionamento, CI/CD, testes, observabilidade e governança** ao longo de todo o ciclo de vida dos pipelines.

---

## 🎯 Objetivos do repositório

* Introduzir os princípios de **DataOps** e a sua aplicação em projetos de dados
* Demonstrar como implementar **CI/CD para pipelines de dados**
* Explorar estratégias de **qualidade e validação de dados**
* Mostrar como observabilidade e lineage aumentam a confiança operacional
* Evoluir pipelines experimentais para soluções produtivas e auditáveis
* Evitar:

  * alterações manuais em ambientes produtivos
  * ausência de testes e validações de dados
  * pipelines sem monitorização ou rastreabilidade
  * dependência excessiva de conhecimento tribal

## 🎬 Demo prática (sessão guiada)

A formação inclui uma **demo end-to-end**, focada em cenários reais:

* Desenvolvimento e versionamento de pipelines
* Automatização de deploys através de CI/CD
* Orquestração com Azure Data Factory
* Execução de validações de qualidade de dados
* Deteção e tratamento de falhas
* Observabilidade e rastreabilidade operacional
* Auditoria e lineage dos dados ao longo do pipeline

O objetivo da demo é mostrar como transformar um pipeline funcional num pipeline **confiável, governado e preparado para produção**, reduzindo riscos operacionais e aumentando a confiança nos dados.
